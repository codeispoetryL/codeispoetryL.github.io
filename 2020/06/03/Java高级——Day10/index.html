枚举类

理解：

  • 类的对象只有有限个，确定的，称此类为枚举类
  • 当需要定义一组常量时，建议使用枚举类
  • 若枚举类中只有一个对象，则可作为单例模式的实现方式

定义：

方式一：jdk 5.0之前，自定义枚举类

方式二：jdk 5.0，可以使用enum关键字定义枚举类

说明：方式二定义的枚举类默认继承于java.lang.Enum类，而不是Object类

常用方法：

  • values()：返回枚举类型的对象数组，该方法可以很便利地便利所有枚举值
  • valueOf(String str)：可以将一个字符串转为对应的枚举类对象，要求字符串必须是枚举类对象的“名字”，否则会报错IllegalArgumentException
  • toString()：返回当前枚举类对象常量的名称

使用enum关键字定义的枚举类实现接口的情况：

方式一：实现接口，在enum类中实现抽象方法
方式二：让枚举类的对象分别实现接口中的抽象方法

注解(Annotation)

概述：

  • jdk 5.0开始，Java增加了对元数据的支持，也就是注解

  • jdk 8中注解的新特性

    • 重复注解：

      • 声明@Repeatable
      • 元注解需要相同

    • 类型注解(Target注解中的两个新增值)：

      • TYPE_PARAMETER：表示该注解能写在类型变量的声明语句中
      • TYPE_USE：表示该注解能写在使用类型的任何语句中

示例：

  • 生成文档相关的注解
  • 编译时进行格式检查的注解
    • @Override：限定重写父类方法，只能用于方法
    • @Deprecated：表示所修饰元素已过时
    • @SuppressWarnings：抑制编译器警告
  • 跟踪代码依赖性，实现替代配置文件功能的注解

自定义注解：

  • 注解声明为@interface
  • 内部定义成员，通常使用value表示
  • 可以指定成员的默认值，使用default定义
  • 若自定义注解没有成员，表明是一个标识作用
  • 如果注解有成员，在使用注解时，需要指明成员的值

元注解：

  • 概述：用于修饰其他注解的注解
  • 使用：
    • @Retention：指定所修饰注解的生命周期。值为SOURCE、CLASS(默认)、RUNTIME，只有声明为RUNTIME的注解，才能通过反射获取
    • @Target：指定所修饰注解可以修饰哪些结构
    • @Documented：指定所修饰的注解类将被javadoc提取成文档
    • @Inherited：被修饰的注解将具有继承性
