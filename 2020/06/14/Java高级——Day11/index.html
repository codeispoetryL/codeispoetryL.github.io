集合

引入

使用数组(Array)存储对象具有一些弊端：

  • 一旦初始化后，长度是确定的，无法修改
  • 数组一旦定义好，其元素的类型是确定的。因此只能操作指定类型的数据
  • 数组中提供的方法有限，对于添加、删除、插入数据等操作，非常不方便，且效率不高
  • 获取数组中实际元素的个数的需求，数组没有现成的属性或方法可用
  • 数组存储数据的特点是有序、可重复的

概述

  • 集合、数组都是对多个数据进行存储操作的结构，简称Java容器。此时的存储，指的是内存层面的存储，不涉及到持久化的存储
  • 集合可分为Collection和Map两种体系
    • Collection接口：单列数据
      • List（动态数组）：元素有序、可重复的集合
        • ArrayList、LinkedList、Vector
      • Set：元素无序、不可重复的集合
        • HashSet、LinkedHashSet、TreeSet
    • Map接口：双列数据，保存具有映射关系的集合
      • HashMap、LinkedHashMap、TreeMap、Hashtable、Properties

Collection接口

常用方法：

  • boolean add(Object obj)：将obj元素添加到集合中
  • int size()：获取元素的个数
  • boolean addAll(Collection coll)：将coll集合中的元素添加到当前的集合中
  • boolean isEmpty()：判断当前集合是否为空
  • void clear()：清空集合元素
  • boolean contains(Object obj)：判断当前集合中是否包含obj。仅比较内容
  • boolean containsAll(Collection coll)：判断coll中的元素是否都在当前集合中
  • boolean remove(Object obj)：移除某个元素
  • boolean removeAll(Collection coll)：从当前集合中移除coll中所有的元素
  • boolean retainAll(Collection coll)：获取两个集合的交集，并返回给当前集合
  • boolean equals(Object obj)：判断当前集合和形参集合是否都相同
  • Object[] toArray()：将集合转换为数组
  • asList(Object[] obj)：调用Arrays类的静态方法，将数组转换为集合。数组使用包装类
  • hashCode()：返回当前对象的哈希值
  • iterator()：返回Iterator接口的实例，用于遍历集合元素

集合元素的遍历：

  • 迭代器：

    • hasNext()
    • next()
    • remove()

  • foreach：

    • Java 5.0提供foreach循环迭代访问集合和数组
    • 遍历不再需要获取集合或数组的长度，无需使用索引访问元素
    • 遍历集合的底层调用iterator完成操作

注意事项：

  • 向Collection接口的实现类的对象中添加数据obj时，要求obj所在类重写equals()
  • 集合对象每次调用iterator()方法都得到一个全新的迭代器对象，默认游标都在集合的第一个元素之前
  • 若未调用nesxt()或在掉用next()方法后紧接着调用remove()方法，则会报错IllegalStateEXception

Collection子接口——List接口

概述：

  • 鉴于数组用来存储数据的局限性，通常使用List替代数组
  • List集合类特点：元素有序、可重复

实现类：

  • ArrayList
    • 是List接口的主要实现类
    • 线程不安全，效率高
    • 底层使用数组存储
    • 默认扩容为原来的1.5倍
  • LinkedList
    • 底层使用双向链表存储，因此对于频繁的删除、插入操作，此类效率更高
    • 因为使用链表存储，所以不存在扩容一说
  • Vector
    • 是List接口的古老实现类
    • 线程安全，效率低
    • 默认扩容为原来的2倍

ArrayList源码分析：

  • jdk 7情况下：
    • 底层创建了长度是10的Object[] 数组elementData
    • 若添加操作导致数组容量不够，则默认扩容为原来的1.5倍，并将数据复制到新数组中
    • 建议开发中使用带参的构造器，即开始就将长度大致定义出来
  • jdk 8情况下：
    • 底层数组初始化为{ }，并没有创建长度为10的数组
    • 第一次调用add()时，底层才创建长度为10的数组
    • 类似于单例模式的懒汉式，延迟了数组的创建，节省了内存

常用方法：

  • void add(int index, Object ele)：在index位置插入ele元素
  • boolean addAll(int index, Collection eles)：从idnex位置开始将eles中的所有元素添加进来
  • Object get(int index)：获取指定index位置的元素
  • int indexOf(Object obj)：返回obj在集合中首次出现的位置。若未找到则返回-1
  • int lastIndexOf(Object obj)：返回obj在集合中最后出现的位置
  • Object remove(int index)：移除指定index位置的元素，并返回此元素
  • Object set(int index, Obnject ele)：设置指定index位置的元素为ele
  • List subList(int fromIndex, int tolndex)：返回从fromIndex到toIndex位置的左闭右开的子集合

Collection子接口——Set接口

概述：

  • Set集合类特点：元素无序，不可重复

实现类：

  • HashSet
    • Set接口的主要实现类
    • 线程不安全，可以存储null值
  • LinkedHashSet
    • 是HashSet的子类
    • 遍历其内部数据时，可以按照添加的顺序遍历
    • 通过给添加的数据设置前后节点来实现记录存入数据的顺序（适合频繁的遍历操作）
  • TreeSet
    • 可以按照添加对象的指定属性，进行排序
    • 要求只能添加相同类的对象
    • 自然排序中，比较两个对象是否相同的标准为：compareTo()返回0，而不是equals()
    • 定制排序中，标胶的标准为：compare()返回0，而不是equals()

Set特性分析：

  • 无序：不等于随机性。存储的数据在底层数据中并非按照数组索引的顺序添加，而是根据数据的哈希值决定
  • 不可重复：相同的元素只能存在一个

元素添加过程（以HashSet为例）：

  1. 向HashSet中添加元素a，首先调用a所在类的hashCode()方法，计算a的哈希值，接着通过某种算法计算出a在底层数组中的存放位置，判断数组此位置上是否已经有元素
  2. 若无其他元素，则a添加成功；若有其他元素b（或以链表形式存在多个元素），则比较a与b的哈希值
  3. 若哈希值不相同，则a添加成功；若哈希值相同，则调用a所在类的equals()方法
  4. 若equals()返回true，则a添加失败；若equals()返回false，则a添加成功

注：对于添加成功的3、4情况而言，与已经存在的指定位置上数据以链表方式存储。jdk 7，a放到数组中，指向原来的元素；jdk 8，原来的元素在数组中，指向a（即七上八下）

注意事项：

  • 向Set中添加的数据，其所在的类一定要重写hashCode()和equals()方法，且两个方法尽可能保持一致性
