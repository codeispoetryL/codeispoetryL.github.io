多线程

概述

程序：为完成特定任务、用某种语言编写的一组指令的集合，即指一段静态的代码，静态对象
进程：是程序的一次执行过程，或是正在运行的一个程序

  • 进程作为资源分配的单位，系统在运行时会为进程分配不同的内存区域（方法区和堆空间）

线程：是一个程序内部的一条执行路径

  • 若一个进程同一时间并行执行多个线程，就是支持多线程的
  • 线程作为调度和执行的单位，每个线程拥有独立的运行栈和程序计数器

并行：多个CPU同时执行多个任务
并发：一个CPU（采用时间片）同时执行多个任务
多线程的好处：

  • 提高应用程序的响应，对图形化界面更有意义
  • 提高计算机系统CPU的利用率
  • 改善程序结构。将即长又复杂的进程分为多个线程，独立运行，便于理解和修改

多线程使用情景：

  • 程序需要同时执行两个或多个任务
  • 程序需要实现一些需要等待的任务时
  • 需要一些后台运行的程序时

线程的创建和使用

方式一：继承Thread类

  • 1.创建一个继承与Thread类的子类

  • 2.重写Thread类的run()

  • 3.创建Thread类的子类的对象

  • 4.通过此对象调用start()

方式二：实现Runnable接口（推荐）

  • 1.创建一个实现了Runnable接口的类

  • 2.实现类去实现Runnable中的抽象方法run()

  • 3.创建实现类的对象

  • 4.将此对象作为参数传递到Thread类的构造器中，创建Thread类的对象

  • 5.通过Thread类的对象调用start()

方式三：实现Callable接口（JDK5.0新增）

  • 与使用Runnable接口相比，Callable新增的功能：

    • 相比run()，call()可以有返回值
    • call()可以抛异常
    • Callable 支持泛型的返回值
    • 需要借助FutureTase类，如获取call()的返回值

  • 使用：

    • 1.创建一个实现Callable的实现类
    • 2.实现call方法，将此线程需要执行的操作声明在call()中
    • 3.创建Callable接口实现类的对象
    • 4.将此Callable接口实现类的对象作为参数传递到FutureTask构造器中，并创建FutureTask的对象
    • 5.将FutureTask的对象作为参数传递到Thread类的构造器中，并创建Thread对象，调用start()
    • 6.获取Callable中call()的返回值（有需要的话再使用，不需要可以不写）

方式四：使用线程池（JDK5.0新增）

  • 引入：经常创建和销毁、使用量特别大的资源，比如并发情况下的线程，对性能影响很大

  • 理解：提前创建好多个线程，放入线程池中，使用时直接获取，使用完放回池中。可以避免频繁的创建销毁、实现重复利用

  • 好处：

    • 提高响应速度。减少了创建新线程的时间
    • 降低资源消耗。重复利用线程池中线程，不需要每次都创建
    • 便于线程管理

  • 使用：

    • 1.提供指定线程数量的线程池
    • 2.执行指定的线程的操作。需要提供实现Runnable接口或Callable接口实现类的对象
    • 3.关闭连接池

Thread类中的常用方法：

  • void start()：启动线程，调用当前线程的run()
  • run()：线程在被调度时执行的操作。通常需要重写此方法
  • String getName()：返回线程的名称
  • void setName(String name)：设置线程的名称
  • static Thread currentThread()：返回当前线程。在Thread子类中就是this，通常用于主线程和Runnable实现类
  • yield()：释放当前CPU的执行权
  • join()：在线程a中调用线程b的该方法，此时线程a就进入阻塞状态，知道线程b执行完后，线程a才结束阻塞状态
  • sleep(long millitime)：让当前线程睡眠指定的时间，在该时间内，线程处于阻塞状态
  • isAlive()：判断当前线程是否存活

线程的调度：

  • 优先级等级：
    • MAX_PRIORITY = 10
    • MIN_PRIORITY = 1
    • NORM_PRIORITY = 5
  • 如何获取和设置当前线程的优先级：
    • getPriority()
    • setPriority(int p)

注意事项：

  • start()方法的作用：
    • 启动当前线程
    • 调用当前线程的run()方法
  • 不能通过直接调用run()的方式启动线程
  • 不能让已经调用start()的线程再次调用start()来启动一个线程，会报错IllegalThreadStartException
  • 高优先级的线程要抢占低优先级线程CPU的执行权。但是，并不意味着只有当高优先级的线程执行完后，低优先级的线程才会执行
  • 开发中，优先选择实现Runnable接口的方式创建线程。因为：
    • 实现的方式没有类的单继承的局限性
    • 实现的方式更适合来处理多个线程共享数据的情况

线程的生命周期

线程的同步

引入：

  • 问题：卖票过程中，出现的重票、错票等线程安全问题
  • 原因：当某个线程操作车票的过程中，尚未操作完成时，其他线程参与进来，也操作车票
  • 解决：当一个线程a操作车票的时候，其他线程不能参与进来。直到线程a操作完成，其他线程才可以开始操作车票
  • 结论：在Java中，通过同步机制，来解决线程的安全问题

同步机制：

  • 方式一：同步代码块

    • 使用：

    • 说明：

      • 操作共享数据的代码，即为需要被同步的代码
      • 共享数据：多个线程共同操作的变量。如：引入中的车票
      • 同步监视器：即锁。任何一个类的对象，都可以充当锁。要求多个线程必须共用同一把锁

  • 方式二：同步方法

    • 概念：如果操作共享数据的代码完整的声明在一个方法中，则将此方法声明为同步的
    • 使用：在方法声明处加synchronized
    • 说明：
      • 同步方法仍然涉及到同步监视器，只是不需要我们显示的声明
      • 同步方法处理继承Thread类的同步监视器为当前类；处理实现Runnable接口的同步监视器为this
      • 非静态的同步方法，同步监视器是this；静态的同步方法，同步监视器是当前类本身

  • 方式三：Lock锁（JDK5.0新增）

优缺点：

  • 好处：同步的方式，解决了线程的安全问题
  • 局限性：操作同步代码时，只能有一个线程参与，其他线程等待。相当于是一个单线程的过程，效率低

两个问题：

  • 线程安全的单例模式之懒汉式
  • 线程的死锁问题
    • 理解：
      • 不同的线程分别占用对方需要的同步资源不放弃，都在等待对方放弃自己需要的同步资源，就行程了线程的死锁
    • 注意事项：
      • 出现死锁后，不会出现异常，不会出现提示，只是所有的线程都处于阻塞状态，无法继续
    • 解决办法：
      • 专门的算法、原则
      • 尽量减少同步资源的定义
      • 尽量避免嵌套同步

线程的通信

涉及到的三个方法：

  • wait()：一旦执行此方法，当前线程就进入阻塞状态，并释放同步监视器
  • notify()：一旦执行此方法，就会唤醒被wait的一个线程。若有多个线程被wait，则唤醒优先级高的线程
  • notifyAll()：一旦执行此方法，就会唤醒所有被wait的线程

说明：

  • 三个方法必须使用在同步代码块或同步方法中
  • 三个方法的调用者必须是同步代码块或同步方法中的同步监视器。否则，会报错IllegalMonitorStateException
  • 三个方法是定义在java.lang.Object类中
